# GitHub Upload Checklist

Use this checklist before making the repository public.

- [ ] Replace placeholder screenshots in `docs/screenshots/` with your own execution evidence.
- [ ] Confirm there are no real passwords, tokens, private keys, kubeconfigs, or `.env` files committed.
- [ ] Update repository URL references after creating the GitHub repo.
- [ ] Run the validation commands from the README.
- [ ] Add a short project description and topics in GitHub repository settings.
- [ ] Add cleanup screenshots or notes for cloud-cost projects.
- [ ] Pin the strongest six repositories on your GitHub profile.

Recommended GitHub topics: `devops`, `cloud`, `aws`, `terraform`, `docker`, `kubernetes`, `cicd`, `github-actions`, `jenkins`, `sre`, `devsecops`.

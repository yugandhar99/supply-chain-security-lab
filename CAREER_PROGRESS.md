# 12-Repo DevOps Career Progression Map

Use these repositories as a story of hands-on growth across DevOps, Cloud, Kubernetes, SRE, DevSecOps, and AI-enabled deployments.

1. **jenkins-java-devsecops-cicd** — CI/CD foundation: I started by building a classic enterprise delivery pipeline for a Java Spring Boot application.
2. **ecs-fargate-terraform-github-actions** — AWS container step: after CI/CD basics, I moved into AWS managed containers using ECS/Fargate and Terraform.
3. **node-cicd-eks-github-actions** — Kubernetes step: next I moved from ECS into EKS and Kubernetes-based delivery using GitHub Actions.
4. **serverless-api-dynamodb-terraform** — Serverless step: I expanded from containers into event-driven AWS serverless architecture.
5. **django-saas-ecommerce-devops** — Application platform step: I worked on a larger real-world style application with database, Docker, Kubernetes, and AWS deployment assets.
6. **cloud-native-monitoring-eks** — Monitoring step: after deployments, I focused on visibility, health checks, and operational readiness.
7. **opentelemetry-observability-lab** — Advanced observability step: I expanded monitoring into traces, metrics, collectors, dashboards, and local SRE labs.
8. **argocd-gitops-home-lab** — GitOps step: I moved deployment control from manual kubectl commands into Git-based desired state management.
9. **progressive-delivery-argo-rollouts** — Release strategy step: after GitOps, I focused on safer releases using canary deployment and rollback workflows.
10. **supply-chain-security-lab** — DevSecOps step: I added supply-chain security practices like image scanning, SBOM generation, and image signing concepts.
11. **jenkins-argocd-vault-devsecops** — Enterprise DevSecOps step: I combined Jenkins, security scanning, secret management, artifact handling, and GitOps deployment.
12. **openai-chatbot-eks-devsecops** — AI-enabled DevOps step: I applied DevOps and DevSecOps practices to an AI chatbot application deployed on Kubernetes.
